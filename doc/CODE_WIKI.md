# Word 文本处理工具 - Code Wiki

## 📋 项目概述

### 1.1 项目定位

**Word 文本处理工具**是一款功能完善的 Word 文档批量处理工具，支持批量读取 `.doc` 和 `.docx` 格式文件，按规则批量替换文本并保持原文档格式（字体、字号、颜色、段落样式等）完全不变。

### 1.2 核心价值

| 特性 | 描述 |
|------|------|
| 批量处理 | 支持同时处理多个 Word 文档 |
| 格式保持 | 替换文本时保持原有格式不变 |
| 智能替换 | 基于类别（项目编号、公司名称等）进行批量替换 |
| 独立运行 | 打包为单个 exe 文件，无需 Python 环境 |

## 📦 核心模块详解

### 3.1 WordProcessor 类

**功能定位**：负责 Word 文档的读取、文本替换和格式保持

**文件位置**：`src/main.py`

#### 3.1.1 类属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `MAX_FILE_SIZE` | int | 50MB | 最大文件大小限制 |
| `MAX_REPLACE_TEXT_LENGTH` | int | 10000 | 最大替换文本长度 |

#### 3.1.2 核心方法

##### `get_word_files(path)`

获取指定路径下的所有 Word 文件

| 参数 | 类型 | 说明 |
|------|------|------|
| `path` | str/Path/list | 文件路径、文件夹路径或文件列表 |

**返回值**：`list` - Word 文件路径列表

**处理流程**：
1. 验证路径存在性
2. 解析路径类型（文件/文件夹/列表）
3. 过滤 `.doc` 和 `.docx` 文件
4. 验证文件大小和访问权限

##### `replace_text_in_paragraph(paragraph, replacements)`

在段落中替换文本，保持格式不变

| 参数 | 类型 | 说明 |
|------|------|------|
| `paragraph` | Paragraph | docx 段落对象 |
| `replacements` | dict | 替换规则字典 {原词: 新词} |

**处理流程**：
1. 按原词长度降序排序（长词优先）
2. 查找匹配文本
3. 调用 `_replace_text_preserve_format()` 执行替换

##### `_replace_text_preserve_format(paragraph, old_text, new_text)`

**核心算法**：替换文本并保持格式

**处理逻辑**：
1. 计算每个 run 的位置范围
2. 定位受影响的 runs
3. 单 run 替换：直接修改文本
4. 多 run 替换：合并格式并删除中间 runs

##### `process_document(doc_path, replacements, output_path, callback=None)`

处理单个 Word 文档

| 参数 | 类型 | 说明 |
|------|------|------|
| `doc_path` | str | 源文档路径 |
| `replacements` | dict | 替换规则 |
| `output_path` | str | 输出路径 |
| `callback` | function | 完成回调 |

**返回值**：`bool` - 处理成功返回 True

### 3.2 ExcelProcessor 类

**功能定位**：负责将编号写入 Excel 文件

**文件位置**：`src/main.py`

#### 3.2.1 核心方法

##### `write_number_to_excel(excel_path, number_text, use_fixed_mode=False, fixed_number="", number_format="", callback=None)`

将编号写入 Excel 文件

| 参数 | 类型 | 说明 |
|------|------|------|
| `excel_path` | str | Excel 文件路径 |
| `number_text` | str | 生成的编号文本 |
| `use_fixed_mode` | bool | 是否使用固定编号模式 |
| `fixed_number` | str | 固定编号内容 |
| `number_format` | str | 编号格式 |

**写入位置**：
- 自动模式：编号写入 E1:F1 合并单元格
- 固定模式：固定编号写入 E1:F1，格式编号写入 E2:F2

### 3.3 WordProcessorGUI 类

**功能定位**：用户界面和交互逻辑

**文件位置**：`src/main.py`

#### 3.3.1 界面结构

```
主窗口
├── 选项卡容器 (QTabWidget)
│   ├── 文本替换选项卡
│   │   ├── 文件选择区域
│   │   ├── 文件列表区域
│   │   ├── 替换规则区域
│   │   ├── 保存设置区域
│   │   ├── 进度显示区域
│   │   └── 操作按钮区域
│   └── 记录编号选项卡
│       ├── 编号模式选择
│       ├── 编号格式设置
│       ├── 固定编号输入
│       ├── Excel文件选择
│       ├── 进度显示区域
│       └── 操作按钮区域
```

#### 3.3.2 核心方法

##### `_create_text_replace_tab()`

创建文本替换选项卡的所有组件

##### `_start_processing()`

启动批量处理（后台线程）

**处理流程**：
1. 验证文件列表和替换规则
2. 禁用开始按钮防止重复点击
3. 创建后台线程执行处理
4. 实时更新进度条和状态

##### `_get_all_replacements()`

获取所有替换规则并转换为字典

##### `_generate_number()`

生成编号，序号自动递增（超过99后变为3位数）

## 🎯 使用流程

### 6.1 文本替换流程

1. **选择文件**：点击"浏览文件"或"浏览文件夹"
2. **管理列表**：通过"+"/"-"按钮或双击管理文件列表
3. **设置规则**：在表格中编辑替换规则
4. **设置路径**：选择保存路径（可选）
5. **开始处理**：点击"开始处理"按钮

### 6.2 记录编号流程

1. **选择模式**：自动编号模式或固定编号模式
2. **设置格式**：输入编号格式（使用 `01` 作为占位符）
3. **选择文件**：选择要写入的 Excel 文件
4. **写入编号**：点击"写入编号"按钮

## 🔒 安全特性

### 8.1 文件安全检查

- **大小限制**：最大 50MB，自动过滤超大文件
- **权限验证**：检查文件读取权限和文件夹写入权限
- **路径规范化**：使用 `resolve()` 获取绝对路径，防止路径遍历攻击

### 8.2 输入验证

- **长度限制**：替换文本最大 10000 字符
- **空值处理**：查找内容为空时规则不生效
- **异常捕获**：全面的 try-except 包裹
