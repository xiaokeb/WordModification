# Word 文本处理工具 - 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
Word 文本处理工具

### 1.2 项目目标
开发一款功能完善的 Word 文档批量处理工具，支持批量读取 .doc 和 .docx 格式文件，按规则批量替换文本并保持原文档的格式（字体、字号、颜色、段落样式等）完全不变。

### 1.3 技术栈
- **编程语言**: Python 3.x
- **GUI 框架**: Tkinter
- **文档处理**: python-docx
- **打包工具**: PyInstaller
- **运行平台**: Windows（无需 Python 环境）

---

## 2. 功能需求

### 2.1 文件选择功能

#### 2.1.1 浏览文件
**功能描述**: 用户可通过文件对话框选择一个或多个 Word 文件

**详细需求**:
- 支持多选文件（按住 Ctrl 或 Shift）
- 文件类型过滤：仅显示 .doc 和 .docx 文件
- 单个文件选择时，路径标签显示文件所在文件夹路径
- 多个文件选择时：
  - 若文件在同一文件夹：显示共同文件夹路径
  - 若文件在不同文件夹：显示"已选择 X 个文件"
- 自动过滤超过 50MB 的文件
- 验证文件可读性权限

#### 2.1.2 浏览文件夹
**功能描述**: 用户可选择包含 Word 文件的文件夹，自动识别其中所有 Word 文件

**详细需求**:
- 自动扫描文件夹内所有 .doc 和 .docx 文件
- 过滤超过 50MB 的文件
- 路径标签显示所选文件夹的完整路径（最后一级文件夹）
- 若文件夹内无 Word 文件，显示警告提示
- 验证文件夹存在性和可访问性

#### 2.1.3 文件列表管理
**功能描述**: 用户可手动管理已选择的文件列表

**详细需求**:
- **添加文件**: 点击"+"按钮，可添加更多文件到列表
- **删除文件**: 
  - 选中文件后点击"-"按钮删除
  - 双击文件直接删除
- **文件列表显示**:
  - 仅显示文件名（不含路径）
  - 无列标题，直接显示文件名
  - 支持滚动查看大量文件
- **路径信息显示**:
  - 绿色标签显示"已选择 X 个文件"
  - 标签后方显示路径信息：
    - 单个文件：显示"路径：文件夹路径"
    - 多个同目录文件：显示"路径：共同文件夹路径"
    - 多个不同目录文件：显示"路径：X 个不同目录"
    - 未选择文件：显示"未选择路径"

### 2.2 替换规则设置功能

#### 2.2.1 规则表格界面
**功能描述**: 采用表格形式管理替换规则，包含三列

**详细需求**:
- **列结构**:
  - 第 1 列：类别（灰色字体，不可编辑）
  - 第 2 列：查找（原词，白色背景，可编辑）
  - 第 3 列：替换（新词，白色背景，可编辑）
- **列宽设置**:
  - 类别列：120px，居中对齐
  - 查找列：350px
  - 替换列：350px
- **行高设置**: 40px
- **字体设置**: Microsoft YaHei UI, 12pt

#### 2.2.2 默认规则
**功能描述**: 系统预设替换规则模板

**详细需求**:
- **项目编号**:
  - 查找：HT 合同编号 - 年份-XM001-01
  - 替换：HT 合同编号 - 年份-XM001-01
- **公司名称**: 空
- **系统名称**: 空
- **人员名称**: 空

#### 2.2.3 规则编辑功能
**功能描述**: 用户可双击单元格编辑替换规则

**详细需求**:
- **编辑触发**: 双击单元格进入编辑模式
- **编辑限制**:
  - 类别列（第 1 列）不可编辑
  - 查找列和替换列可编辑
- **编辑操作**:
  - 按 Enter 键保存编辑
  - 点击其他地方保存编辑
  - 按 Escape 键取消编辑
- **空规则处理**:
  - 查找内容为空时，该规则不生效
  - 替换内容为空时，表示删除所有匹配的查找内容

#### 2.2.4 规则重置功能
**功能描述**: 一键重置为默认规则

**详细需求**:
- 点击"重置为默认值"按钮
- 清空所有自定义规则
- 恢复默认规则列表

### 2.3 保存设置功能

#### 2.3.1 自定义保存路径
**功能描述**: 用户可指定处理后文件的保存位置

**详细需求**:
- 点击"浏览"按钮选择保存文件夹
- 验证文件夹存在性
- 验证写入权限（通过创建测试文件）
- 选择后显示完整路径
- 显示状态提示："已选择自定义保存路径"

#### 2.3.2 自动保存规则
**功能描述**: 未指定保存路径时，按规则自动保存

**详细需求**:
- 默认保存路径：原文件所在目录的"修改版"子文件夹
- 文件名规则：`原文件名 - 修改版.docx`
- 自动创建输出文件夹（若不存在）

### 2.4 批量处理功能

#### 2.4.1 处理启动
**功能描述**: 用户点击"开始处理"按钮启动批量替换

**前置条件**:
- 已选择至少一个 Word 文件
- 已设置至少一条替换规则（查找内容不为空）

**详细需求**:
- 验证前置条件
- 禁用"开始处理"按钮（防止重复点击）
- 启动多线程处理（避免界面卡顿）

#### 2.4.2 处理过程
**功能描述**: 后台处理文件，实时显示进度

**详细需求**:
- **进度显示**:
  - 进度条：0-100% 实时进度
  - 文字提示："正在处理：文件名 (当前/总数)"
- **文件处理**:
  - 逐个处理选中的文件
  - 保持原文档格式不变
  - 支持 .docx 格式处理
- **线程安全**: 使用 `root.after` 更新 UI

#### 2.4.3 处理完成
**功能描述**: 处理完成后显示结果

**详细需求**:
- 启用"开始处理"按钮
- 显示处理结果统计：
  - 成功数量/总数量
  - 全部成功：显示"修改完成，已保存"
  - 部分成功：显示警告提示
- 进度标签显示"处理完成"

### 2.5 清空功能

**功能描述**: 一键清空所有设置

**详细需求**:
- 清空文件列表
- 清空保存路径
- 重置替换规则为默认值
- 重置进度条和进度标签
- 重置所有状态标签

## 3. 安全性需求

### 3.1 文件安全检查

#### 3.1.1 文件大小限制
- 最大文件大小：50MB
- 超过限制的文件自动过滤
- 显示警告提示

#### 3.1.2 文件存在性验证
- 验证文件是否存在
- 验证文件是否可访问
- 验证文件读取权限

#### 3.1.3 路径安全
- 路径规范化处理（防止路径遍历攻击）
- 使用 `resolve()` 获取绝对路径
- 验证输出路径合法性

### 3.2 输入验证

#### 3.2.1 替换文本长度限制
- 最大替换文本长度：10000 字符
- 超过限制的替换规则自动跳过

#### 3.2.2 空输入处理
- 查找内容为空：规则不生效
- 替换内容为空：删除匹配内容（需确认）

### 3.3 异常处理

#### 3.3.1 全面异常捕获
- 所有文件操作包含 try-except
- 防止恶意输入导致程序崩溃
- 显示友好的错误提示

#### 3.3.2 权限验证
- 文件读取权限验证
- 文件夹写入权限验证（通过测试文件）
- 无权限时显示错误提示

## 8. 数据流图

### 8.1 文件选择流程
```
用户操作 → 选择文件/文件夹 → 验证文件 → 过滤文件 → 更新列表 → 显示路径
```

### 8.2 替换处理流程
```
用户点击开始 → 验证条件 → 获取规则 → 多线程处理 → 逐个文件处理 → 更新进度 → 完成提示
```

### 8.3 文件处理流程
```
读取文档 → 遍历段落 → 查找匹配 → 格式保持替换 → 遍历表格 → 保存文档
```

## 9. 错误处理

### 9.1 文件相关错误
- 文件不存在：显示错误提示
- 文件过大：显示警告，自动过滤
- 无访问权限：显示错误提示
- 格式不支持：跳过处理

### 9.2 路径相关错误
- 路径不存在：显示错误提示
- 无写入权限：显示错误提示
- 路径非法：拒绝操作

### 9.3 处理相关错误
- 文档损坏：跳过并记录
- 替换失败：继续处理其他文件
- 保存失败：显示错误提示
